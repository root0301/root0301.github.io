If the last word does not exist, return 0.

Note: A word is defined as a character sequence consists of non-space characters only.

For example,
Given s = "Hello World",
return 5.

Java与线程

发表于 2017-07-03 | 分类于 Java虚拟机

线程的实现

​ 线程是比进程更轻量级的调度执行单位，线程的引入可以把一个进程的资源分配和执行调度分开。

​ 对于Java的Thread类，它和大部分的API有所不同，它的所有的关键方法都试声明为Native的。

​ 实现线程主要有3中方式：内核线程实现、用户线程实现、用户线程＋轻量级进程混合实现

内核线程实现

​ 内核线程（Kernel-Level Thread）直接由操作系统内核支持的线程，由内核来完成线程切换。内核通过操纵调度器对线程进行调度，将线程任务映射到各个处理器上。程序一般是使用内核线程的高级接口——轻量级进程（Light-Weight Process），这才是我们通常意义上所讲的线程，LWP和KLT的的关系是1:1。

​ 优点：每个LWP都是一个独立的线程调度单元，不会发生阻塞

​ 缺点：线程操作都需要系统调用，代价昂贵，需要在用户态和内核态之间频繁切换。

用户线程实现

​ 广义上来讲，一个非内核线程即是用户线程，LWP也可以说是用户线程，但其许多操作需要进行系统调用，效率收到限制。狭义上的用户线程是完全建立在用户空间的线程库上，线程操作快速且低耗。但是线程的创建、切换和调度都是需要考虑的问题，诸如如何处理阻塞、多处理器系统中如何将线程映射到其他处理器上这些问题将会很难解决，使用用户线程实现的程序一般都比较复杂。

用户线程＋轻量级进程混合实现

​ 用户进程仍是完全建立在用户空间上，降低创建、切换、析构等操作的耗能，同时以轻量级进程作为用户线程和内核之间的桥梁，因此可以使用内核提供的线程调度功能和处理器映射，降低了整个进程被阻塞的危险。

Java线程的实现

​ 在JDK1.2之前，是基于称为“绿色线程”的用户线程来实现的，而在JDK1.2中，线程模型替换为基于操作系统的原生线程模型。因此在目前的JDK版本中操作系统的线程模型很大程度上影响了Java虚拟机的线程映射方式。

Java线程调度

协同式调度

​ 线程执行时间由线程本身来控制，线程执行完之后通知系统切换线程，实现简单，但容易造成程序阻塞。

抢占式线程调度

​ 每个线程由系统来分配执行时间，线程的执行时间是可控的，不会出现程序阻塞的问题。虽然线程调度是系统自动完成，但可以通过设置优先级来给某些线程多分配一点执行时间。Java语言一共设置了10哥级别的线程优先级，对于两个同时处于ready状态的线程，高优先级的线程先被执行。但是程序优先级并不是很靠谱，因为Java的线程是通过映射到系统的原生线程上来实现的，最终的线程调度还是取决于操作系统，操作系统的优先级不见得会与Java设置的优先级一一对应。对于线程优先级比Java少的系统，就会出现几个优先级相同的情况，多的话只需在中间留下一些空位即可。还有一种情况就是线程的优先级可能会被系统自行改变，例如Windows中的优先级推进器。

线程的状态

​ 创建、等待、运行、阻塞、结束
